WHAT THE VAXSTATION IS
##: KEYWORD "Overview" :##
##: BLANK :##
The VAXstation 100 system expands the use and convenience of the
VAX/VMS operating system by providing each VAXstation user with as
many terminals (simulated on the VAXstation screen) as needed. 

##: INDEX "Overview of VAXstation" :## 

Each of these simulated terminals is connected to the VAX/VMS host and
can run VAX/VMS processes independently of the others.  
##: BLANK :##
Control of these simulated terminals is simple and straightforward:
they are created, moved, and discarded by pointing and by using menus.
By using the VAXstation 100 system, you can view or manage several
related activities simultaneously. For example, you can: 
##: BLANK :##
##: LM 60 :##
##: INDENT -20 :##
##: SET RENDITION BOLD :##~.^##: SET RENDITION NO BOLD :## Examine 
a listing while editing the source file.
##: BLANK :##
##: INDENT -20 :##
##: SET RENDITION BOLD :##~.^##: SET RENDITION NO BOLD :## Start 
one task and follow its progress while interacting with another.
##: LM 30 :##


CHAPTER 1:  AN OVERVIEW
##: BREAK :##
What the VAXstation Is (cont)
##: BLANK 2 :##
The VAXstation system does not change the behavior of the operating
system.  Once a simulated terminal has been created on the VAXstation
100 system, you can use it just as you use a real (logged-in)
terminal. The operating system is neither changed nor simplified by
the VAXstation system, but it may be made more convenient. 
##: BLANK :##
If your work at a video terminal involves you explicitly with the
VAX/VMS operating system, you can continue the same involvement when
using a VAXstation system. And if you rely on software that shields
you from the need to issue explicit VAX/VMS commands in response to
the DCL dollar sign prompt ($), those software shields can remain in
place.

##: INDENT -24 :##
Surfaces Best Suited For Mouse Operation
##: KEYWORD "Mouse surfaces" :##
##: BLANK :##
For best results, use the mouse on a surface like paper or cardboard.
Although the mouse operates satisfactorily on almost any horizontal
surface, a surface with more friction makes the mouse easier to
operate.  Surfaces like glass, Formica, and vinyl offer less friction
and require more attention to the mouse. 
##: BLANK :##
The direction of the mouse doesn't rotate the arrow.  Thus, if you
pick up the mouse and move it around, the arrow doesn't follow the
direction of the mouse. 
##: BLANK :##

Care of the Mouse
##: KEYWORD "Mouse care" :##
##: BLANK :##
Proper care can extend the life of this delicate electromechanical
device. Avoid using the mouse on dusty or ash-covered surfaces, and
always keep the mouse and its tail dry.

##: KEYWORD "Printer" :##
##: BLANK :##

##: INDEX "Printer" :##

You can print an object that is displayed on the screen if you have
the printer option.  To print an object, select the "Print" menu item
from a menu for a document, terminal, or viewport.

hourglass indicating "Please wait."  The pointer then returns to its

or viewport that you selected.

THE DEC GRAPHICS DATA PAD
##: KEYWORD "Data pad" :##
##: BLANK :##
The DEC Graphics Data Pad is an option with your VAXstation system.

##: INDEX "Graphics data pad", 
"Data pad, graphics" :##
       
This graphics data pad provides the VAXstation system with highly accurate 
positioning information that you can use for entering graphics data.
##: BLANK :##
The DEC Graphics Data Pad has a handheld positioning device (the puck)
containing five push buttons and a transparent window that has a pair
of crosshairs. 

##: BLANK 19 :##
##: OVERLAY VSTA$HILIB:DATAPAD.FNT AT 30,290 :##

##: BREAK :##
The Graphics Data Pad (cont)
##: BLANK 2 :##
The data pad requires proper intialization under the control of an
application program.  When the pad is initialized, you can position the
crosshairs on the data pad surface (while, for example, tracing an
image mounted on that surface). 

When you use the data pad with your VAXstation system, the screen 
displays high-resolution of the cursor position on the data pad
surface. 
##: BLANK :##

A microprocessor located in the base of the data pad controls the graphics
data pad. An etched grid below the data pad's surface senses
electromagnetic pulses generated by a coil within the puck and
computes the puck X and Y axis locations 200 times a second with
0.0254 mm (0.001 inch) resolution.

JOB DEFINITION AND CONTROL
##: BLANK :##
##: KEYWORD "Job definition" :##
Once authorization is accomplished, you can create the VAX/VMS jobs
you need. You can arrange terminals in the work area, covering
some jobs and displaying others, just as you might arrange your desk. 
You can clear the work area by logging out jobs or by temporarily
storing jobs in the utility area at the right. 
##: BLANK :##
You can store up to nine jobs in the utility area at the right.

##: INDEX "Number of jobs",
"Maximum jobs" :##

However, VAXstation imposes no limit to the number of jobs that you 
can stack in the work area.
##: BLANK :##
Like a desk top, however, you pay a price for clutter in the work area
of the VAXstation. The more jobs you stack in the work area, the more
time it takes you to find an object. Further, you may encounter job
limits imposed by VAX/VMS or by your system manager.

THE DEC TERMINAL
##: KEYWORD "DEC terminal" :##
##: BLANK :##
The DEC terminal as simulated by the VAXstation system behaves for
most purposes like the real thing--that is, a VT100 with the
advanced video option (AVO), but without an alternate character ROM or
the graphics processing option. 

##: INDEX "DEC terminal characteristics" :## 

Some similarities include:  (approximate) screen size, number of
lines, screen background, ANSI/VT52, wrap-around, new line, Answerback
Message, and response to escape sequences. 
##: BLANK :##
In contrast to the real VT100 terminal, the simulated DEC terminal
allows only 80 columns, and operates only in "jump scroll" mode.
Further, transmit and receive speeds, interlace, and parity
specifications have no meaning to the simulated terminal.  Thus, the
simulated DEC terminal has no controls for these settings. 
##: BLANK :##

procedures for setting characteristics.  These differences occur
because the simulated VT100 terminal has no physical embodiment (no
local hardware storage mechanism to preserve its settings during a
session or from session to session, and no permanently attached
keyboard).  The VAXstation system handles these differences as
